# ERP 系統現代化與外掛開發備忘錄 (Developer Notes)

本文件用於記錄系統架構升級、功能拓展想法及安全性防護策略，以便後續開發追蹤。

---

## 📌 1. 系統架構升級：中繼資料庫 (Relay Database) + API 接口模式
為解決傳統 Visual FoxPro (VFP) ERP 的 DBF 檔案資料庫在效能、安全與現代化整合上的限制，我們規劃採用中繼資料庫架構：

*   **讀寫分離設計**：
    - **寫入端 (ERP)**：公司員工繼續正常使用舊的 ERP 桌面軟體，資料如常寫入 `.DBF` 檔案。
    - **資料同步橋樑**：撰寫輕量背景同步程式（Node.js 或 PowerShell），實時/定時將 DBF 的新資料拋轉至 SQL 資料庫。
    - **讀取/外掛端**：所有的外掛服務直接對接並讀取 SQL 資料庫。
*   **預計選用資料庫**：PostgreSQL 或 MS SQL Server Express (免費且效能優越)。
*   **API 接口層**：
    - 由 Express.js (`take_server.js` 擴充) 統一封裝資料庫查詢，提供安全、跨平台（Bot、Web）共用的 HTTP API 接口。

## 📷 2. 新功能規劃：Telegram 相機實時條碼掃描
實現手機即時條碼掃描，做為隨身盤點與查詢工具：

*   **技術方案：Telegram Mini App (方案 A)**
    - **運作模式**：在 Telegram 對話框內提供「開啟相機」按鈕，直接彈出內嵌 Web 視窗。
    - **前端實作**：網頁載入開源 JavaScript 條碼庫（如 `html5-qrcode`），調用手機鏡頭進行實時（0.1s）條碼解析。
    - **資料傳遞**：透過 Telegram WebApp SDK 傳回 Bot 或直接打 API 查詢 SQL 資料庫，自動關閉 Web 視窗並發送查詢報表。

## 🛡️ 3. 外掛隱蔽性與防衝突設計 (Evasion & Anti-Locking)
為避免 ERP 廠商在進線維護（重整系統、重建索引、更新結構）時，因為我們的外掛檔案鎖定而引發衝突或被發現，設計以下防範機制：

*   **檔案鎖定防範**：
    - **非持久唯讀連線**：同步程式對 DBF 進行連線時，採用「隨開隨關」讀取方式，嚴禁長期掛載鎖定。
    - **「一鍵暫停」指令**：在 Telegram 機器人端實作 `暫停同步` / `開始同步` 指令。廠商進行維護前，使用者可透過手機一鍵釋放所有 DBF 的檔案鎖定；維護完成後一鍵重啟。
*   **系統重整復原能力**：
    - 所有程式碼與設定腳本完全託管於私有 Git 倉庫 (GitHub)。
    - 若伺服器重灌或環境被洗掉，可於 5 分鐘內透過 Git 拉回專案並執行 `restart_services.ps1` 快速重建。
*   **背景進程偽裝**：
    - 必要時，將背景的 Node.js 執行進程封裝並偽裝成 Windows 系統服務名稱（如 `Windows_Sync_Helper`）以隱匿能見度。
