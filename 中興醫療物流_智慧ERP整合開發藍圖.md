# 中興醫療物流：智慧 ERP 整合開發藍圖

本藍圖整併了針對現行系統的「外掛式自動化升級藍圖」與未來的「全模組化智慧 ERP 終極系統」。核心設計專注於輕量化效能、台灣本土化財稅分流，以及個人專屬 AI 遠端遙控，旨在徹底解放地端算力，建立鐵壁般的商業數據基地。

---

## 📅 第一階段：舊型 VFP 系統外掛防禦與過渡架構

### 1. 唯讀防護與資料夾沙盒隔離
針對現行的 Visual FoxPro (VFP) 舊系統，採用「不干擾、鬆耦合、資料夾隔離」策略，絕對不修改底層代碼：
*   **唯讀防護與亂碼隔離**：對 VFP 原始資料庫僅進行唯讀讀取，防範並行寫入鎖死；統一在局外人層級進行轉碼清洗。

| 磁碟代號 | 規劃用途 | 安全防護機制（沙盒隔離） |
| :--- | :--- | :--- |
| **C 槽 (系統碟)** | Windows 作業系統、必備核心軟體。 | 不存放任何業務資料與自行開發的代碼。 |
| **D 槽 (資料碟)** | 中央資料庫、正式上線外掛、AI 開發沙盒。 | 採用實體與邏輯隔離。包含 `01_唯讀備份`、`02_AI_測試沙盒`、`03_AI_正式上線`、`04_PostgreSQL`。 |

---

### 2. 資料同步與前線機器人 (Bot 1~4) 權限
*   **夜間離峰同步**：半夜自動將 VFP 的商品資料、庫存版次抽離寫入 PostgreSQL。
*   **多通道獨立儲存**：廢除 TXT 日誌，改用 PostgreSQL 作為終極容器，確保 ERP 業務與 AI 智庫互不干擾。

| 機器人編號 | 使用者角色 | 可存取資料權限 (由資料庫過濾) |
| :--- | :--- | :--- |
| **Bot 1** | 主管 / 老闆專用 | 完整權限：歷史成交價、最新底價、庫存版次。 |
| **Bot 2** | 外勤業務人員 | 受限權限：歷史成交價、當前庫存（自動遮蔽利潤）。 |
| **Bot 3 / 4** | 內勤與工位看板 | 任務權限：庫存盤點、採購待辦事項打勾清單。 |

## 🚀 第二階段：全模組化智慧 ERP 終極系統

### 1. 系統底層架構與地端部署
*   **心臟中樞 (NAS 伺服器)**：API、資料庫及監聽程式架設於內網 NAS，配置自動排程優化。
*   **終端網頁與行動自適應**：前線電腦免安裝直接透過瀏覽器作業；手機端自適應 UI，支援直接呼叫 OCR 掃描。
*   **雙層角色權限 (RBAC) 與 VPN**：嚴格劃分 Admin 與 Staff；整合 Tailscale 加密隧道實現遠端安全查帳。

### 2. 核心業務功能與本土化財稅

| 模組分類 | 核心機制 |
| :--- | :--- |
| **基本資料與庫存防禦** | 商品價格分級連動；手機 OCR 單據提取；FEFO (先到期先出) 強制扣庫鎖定；欠貨單儀表板提醒。 |
| **本土化憑證與分流報表** | 全型態發票標記；月底自動匯出 Excel 雙頁籤 (雲端備查 / 實體紙本 Key-in 專區)；帳齡分析催收雷達。 |
| **硬體中介路由** | 動態綁定印表機；帳單預設 Epson LQ-690C 點陣機 (含毫米級微調)，報價單預設 Canon G7000 彩色輸出；一鍵產生雙重版型 PDF。 |

### 3. 專屬 Telegram AI 遙控副手
*   **Chat ID 硬核鎖定**：綁定專屬 ID，非此帳號發送之訊息一律拒絕回應。
*   **自然語言雙向對話**：串接 Gemini API，支援遠端即時查帳 (如「查某客戶 A 級價」) 與語音/文字抄單建檔 (如「出 10 箱貨」自動產生草稿單)。

## 🛠️ 第三階段：實戰開發與排雷策略

1.  **基礎地基驗證 (PoC)**：優先建立資料庫結構、極簡主檔與進出貨數字扣減，確保舊資料乾淨匯入。
2.  **防禦裝甲與實體連動**：加入 FEFO 與價格分級防呆邏輯，設定固定 IP 並進行點陣機套表印刷與公差校正。
3.  **自動化大腦掛載**：實作會計師 Excel 分流匯出、首頁戰情儀表板，最後對接 Telegram 機器人 Webhook。
4.  **營運核心警示**：所有由 Telegram 自動生成的出貨、進貨單據，初始狀態一律鎖定為「待審核草稿」，嚴禁未經人工覆核直接轉為正式單據，確保商業安全。
