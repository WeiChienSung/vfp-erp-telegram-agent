==================================================
        ERP 隨身查與盤點助手 (Telegram Bot) 外掛說明書
==================================================

本外掛包含「ERP 價格與庫存隨身查」、「行動條碼掃描盤點助手」與「客戶歷史售價查詢」，專為醫療器材系統 (Z:\nmedi) 的資料庫管理設計。
本系統採零依賴 (Zero Dependency) 架構，無須安裝額外 npm 套件即可運作。

--------------------------------------------------
一、 系統檔案結構
--------------------------------------------------
本專案已完全整合收納至以下本機路徑。為維護安全隱私，本系統不在 Z 槽 (NAS) 留下任何自訂程式備份。

【目錄結構】：
C:\agy_Add_on\
├── run_bot.vbs                (相容啟動代理腳本，防報錯)
└── ERP 隨身查與盤點\

   C:\Users\5Vce8\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\ERP_隨身查.vbs
2. 啟動原理：
   當 Windows 使用者登入時，系統會自動執行該 VBS，它會以完全隱形 (無視窗) 的方式同時呼叫：

三、 智慧型「缺貨警報與主力低存推播」運作機制 🔔
--------------------------------------------------
為了避免低庫存品項過多導致的 Telegram 訊息轟炸，系統採用智慧分流與狀態偵測技術：

1. 🔴 **即時斷貨偵測（適用於所有正常販售的商品）**：
   系統在設定的時間點（預設 08:30、17:00）會自動比對當前與上次的庫存快照（`stock_snapshot.json`）。**只有在商品的庫存「從大於 0 剛好變成 0（斷貨）」的瞬間，才會觸發 Telegram 警報**。已停產、停售的商品會自動被過濾，不造成打擾。
2. 🟡 **主力商品低庫存警報**：
   如果您有特別關心、暢銷、高單價的商品，請將商品編號填入 `config.json` 的 `priorityProducts` 陣列中。系統會在定時推播時，針對這些商品進行低庫存水位的警報提示（當庫存 <= `lowStockThreshold` 時發送警報）。

四、 視覺化設定後台 (http://lvh.me:3000/config) 🛠️
--------------------------------------------------
您無須再手動用記事本修改 config.json，系統已提供一個精美的本機視覺化設定後台：

1. 開啟方式：
   在安裝此程式的本機電腦瀏覽器中，輸入：http://lvh.me:3000/config (或 http://localhost:3000/config)
   *註：因 Telegram 官方會阻擋 localhost 連結，使用 lvh.me 網址可以從 Telegram 對話中直接點選開啟。*
2. 安全保護機制 (雙重防護鎖) 🛡️：
   - 【本地設定防護鎖】：為防止 Telegram Bot Token 等敏感性憑證外洩，設定後台網頁與對應的 `/api/config` 讀寫 API 僅能於本機電腦 (lvh.me / 127.0.0.1 / localhost) 開啟。任何經由外網 Localtunnel 連結嘗試存取設定網頁的請求都會被系統自動拒絕並回傳 403 Forbidden。
   - 【外網 API Token 金鑰鎖】：當業務從手機點選 Telegram 推送的「行動盤點」網址時，網址結尾已自動附加一組隨機的安全認證金鑰。手機網頁載入時會自動將其暫存至瀏覽器的 sessionStorage 中，並在隨後的廠商與庫存 API 請求中自動發送。後台如果偵測到連線來自非本機（外網），則強制要求此金鑰匹配，任何未帶金鑰的惡意掃描均會被攔截並回傳 401 Unauthorized，保證資料安全。
3. 功能亮點：
   - 【基本設定】：視覺化設定資料庫路徑、拉動滑桿調整「低庫存警戒水位」。
   - 【排程管理】：直接新增、刪除每日定時智慧推播的時間點。
   - 【機器人多槽位配置 (Bot Slots)】：
     * 可以自行新增/刪除多個 Telegram 機器人。
     * 自訂每個機器人的顯示名稱 (如：宋為謙、查價助手)。
     * 設定各機器人 Token 與管理已授權聊天室 ID (支援快速新增/刪除)。
     * 自由勾選每個機器人各自要啟用的四大功能：【查價 (Query)】、【歷史售價 (History)】、【盤點 (Take)】、【庫存推播 (Push)】。
   - 【主力商品管理】：直覺的商品搜尋介面，可一鍵加入「主力列管商品」清單。支援含有雙引號規格（如 2"、3" 等英吋）的商品安全加入。

五、 搜尋引擎優化與中文同音異體字自動對應 🔍
--------------------------------------------------
為了提供最佳的查價與盤點檢索體驗，本系統的搜尋引擎內建了以下進階最佳化技術：

1. 中文同音異體字/簡繁自動對應：
   系統在背景進行比對時，會自動將搜尋詞與商品資料庫中的常見異體字、簡體字進行雙向標準化映射。不論您或建檔人員寫的是哪一個，都能一次找出所有寫法相同的商品：
   - 溼 ↔ 濕 (例如：輸入「克淋溼」或「克淋濕」都能查到相同結果)
   - 臺 ↔ 台
   - 綿 ↔ 棉
   - 雙 ↔ 双
   - 簡繁映射：國↔国、醫↔医、藥↔药、針↔针、紙↔纸。

2. 醫療品牌與複詞自動拆分 (AND 交集搜尋)：
   系統內建了常規品牌與品名複詞字典。當輸入「3M大白」或「華新大白」時，程式會自動將其拆分為「3M」與「大白」進行交集 AND 搜尋，無需手動輸入空白鍵。

六、 客戶專屬歷史售價查詢 (防報錯價神器) 💰
--------------------------------------------------
為了防止業務在客戶現場報錯價格，系統特別提供歷史成交價即時查詢功能：

1. 操作方式：
   在 Telegram 機器人對話框中，直接輸入「客戶關鍵字 商品關鍵字」（順序不拘，例如：`大白 小太羊` 或 `小太羊 網狀`）。
   *註：此功能需在設定後台勾選啟用該 Slots 的【歷史售價 (History)】方可運作。*
2. 系統動作：
   - 系統會自動比對 CUST.DBF，找出最符合的客戶（如 `小太羊藥局`）。
   - 接著採用「逆向分塊緩衝讀取」技術，在 1.3 GB 的 `ougo1.dbf`（銷貨明細）中由新到舊快速檢索。
   - 抓出該客戶「最近三次購買該商品」的日期、商品完整品名、數量（含單位）及當時的實際成交單價。
3. 效能優化：
   系統僅讀取檔案末端最新資料，匹配滿 3 筆即自動終止，即使在 100Mbps 慢速網路共用磁碟下，查詢也能在 0.2 秒內完成，不影響 ERP 伺服器效能。

七、 2026-05-26 系統核心與強健度升級 🛡️
--------------------------------------------------

在本次升級中，系統針對極端環境（網絡不穩、ERP獨佔鎖定、外網掃描）與使用者搜尋體驗進行了全方位重構：

1. 🔍 中英數混合免空格智慧搜尋：
   - 搜尋引擎重構分詞演算法：當輸入「3m1吋膚」或「針頭23g」等無空格混合詞時，系統會自動在英文、數字（支援 2.5 等浮點數）與中文字的邊界進行拆分，交集 AND 搜尋，不再因忘記打空白鍵而找不到商品。
   - 引入搜尋打分機制：緊湊完全匹配給予 100 分，分詞 AND 匹配給予 10 分，結果以得分降序排序，確保最精確的結果永遠排在最前面。

2. 📐 常用單位自動相容映射：
   - 全面相容「寸 ↔ 吋」及「尺 ↔ 呎」，如使用者輸入「1寸」可自動標準化匹配 ERP 資料庫中的「1吋」，大幅減輕手動鍵入單位的負擔。

3. 💰 客戶歷史查詢優化（防干擾）：
   - 對於長度為 1 的單個中文字（如「膚」、「棉」），限制不能模糊匹配客戶全名，必須完全符合客戶代號或簡稱，防止普通查價誤判。
   - 若匹配出的客戶數大於 3 個（如輸入「藥局」、「診所」等通用詞），則系統會判定此非特定客戶查詢，自動降級為一般商品查價。

4. ⚙️ 盤點防檔案損毀鎖定（PowerShell 參數化）：
   - [take_server.js](file:///C:/agy_Add_on/ERP%20%E9%9A%A8%E8%BA%AB%E6%9F%A5%E8%88%87%E7%9B%A4%E9%BB%9E/take_server.js) 與 [append_take.ps1](file:///C:/agy_Add_on/ERP%20%E9%9A%A8%E8%BA%AB%E6%9F%A5%E8%88%87%E7%9B%A4%E9%BB%9E/append_take.ps1) 完全解耦，不再寫死 Z 槽路徑，而是由伺服器動態取得目錄以 -DbDir 傳入。
