==================================================
         ERP 隨身查與盤點助手 (Telegram Bot) 外掛說明書
==================================================

本外掛包含「ERP 價格與庫存隨身查」、「行動條碼掃描盤點助手」、「客戶歷史售價查詢」與「欠貨後送紀錄系統」，專為醫療器材系統 (Z:\nmedi) 的資料庫管理設計。
本系統採零依賴 (Zero Dependency) 架構，無須安裝額外 npm 套件即可運作。

==================================================
⚠️ 專案最高指導原則
==================================================
1. 【唯讀】核心 ERP 系統：本外掛系統的所有功能僅能對原始 ERP 資料庫 (DBF) 檔案進行資料讀取查詢與快取。
2. 嚴禁寫入 Z 槽原始 ERP 系統：盤點上傳結果僅能寫入特定的盤點暫存資料檔（如 `take.dbf` 及 `take1.dbf`），**嚴禁且絕對不可寫入、修改、刪除或覆蓋 Z 硬碟（ZRP 系統）中的任何常規 ERP 資料表或核心系統檔案**，以確保原始 ERP 系統 the 資料安全與運作穩定性。

三、 智慧型「即時斷貨告警與主力低存推播」運作機制 🔔
--------------------------------------------------
為了避免低庫存品項過多導致的 Telegram 訊息轟炸，系統採用模組化狀態比對與過渡狀態機偵測技術：

1. 🔴 **一分鐘即時斷貨偵測（適用於所有正常販售的商品）**：
   背景程式每 60 秒會自動檢測商品庫存修改時間。若有異動，系統會比對最新庫存與 `stock_snapshot.json` 快照。**只有當商品的庫存「從大於 0 剛好變為小於等於 0（斷貨）」的瞬間，才會觸發 Telegram 即時警報**。已停產、停售或品名中含贈品、樣品、測試等字樣的商品會自動過濾忽略，且下班時間（晚上 9:00 後）由於庫存無變動，系統自然保持 100% 靜音。
2. 🟡 **主力商品低庫存警報**：
   如果您有特別關心、暢銷的主力商品，可至後台將其編號加入「列管主力商品」陣列中。當這些列管商品的庫存跌破警戒水位（預設 5 個）的瞬間，系統也會即時推播低存預警，供同仁提早叫貨。
3. 💬 **一鍵查詢商品歷史成交**：
   推播的斷貨警報訊息中，商品品名會以粗體放大優先顯示（便於新人閱讀）。訊息下方會自動附加 `[🔍 查詢此商品歷史成交價]` 快捷按鈕，點擊後機器人會自動向本機銷貨歷史庫檢索並回傳該商品最近 3 次的成交客戶、數量及單價。

四、 視覺化設定後台 (http://lvh.me:3000/config) 🛠️
--------------------------------------------------
您無須再手動用記事本修改 config.json，系統已提供一個精美的本機視覺化設定後台：

1. 開啟方式：
   在安裝此程式的本機電腦瀏覽器中，輸入：http://lvh.me:3000/config (或 http://localhost:3000/config)
   *註：因 Telegram 官方會阻擋 localhost 連結，使用 lvh.me 網址可以從 Telegram 對話中直接點選開啟。*
2. 安全保護機制 (雙重防護鎖) 🛡️：
   - 【本地設定防護鎖】：為防止 Telegram Bot Token 等敏感性憑證外洩，設定後台網頁與對應的 `/api/config` 讀寫 API 僅能於本機電腦 (lvh.me / 127.0.0.1 / localhost) 開啟。任何經由外網 Localtunnel 連結嘗試存取設定網頁的請求都會被系統自動拒絕並回傳 403 Forbidden。
   - 【外網 API Token 金鑰鎖】：當業務從手機點選 Telegram 推送的「行動盤點」網址時，網址結尾已自動附加一組隨機的安全認證金鑰。手機網頁載入時會自動將其暫存至瀏覽器的 sessionStorage 中，並在隨後的廠商與庫存 API 請求中自動發送。後台如果偵測到連線來自非本機（外網），則強制要求此金鑰匹配，任何未帶金鑰的惡意掃描均會被攔截並回傳 401 Unauthorized，保證資料安全。
3. 功能亮點：
   - 【基本設定】：視覺化設定資料庫路徑、拉動滑桿調整「低庫存警戒水位」。
   - 【排程管理】：直接新增、刪除每日定時智慧推播的時間點。
   - 【機器人多槽位配置 (Bot Slots)】：
     * 可以自行新增/刪除多個 Telegram 機器人。
     * 自訂每個機器人的顯示名稱 (如：宋為謙、查價助手)。
     * 設定各機器人 Token 與管理已授權聊天室 ID (支援快速新增/刪除)。
     * 自由勾選每個機器人各自要啟用的功能：【查價 (Query)】、【歷史售價 (History)】、【盤點 (Take)】、【庫存推播 (Push)】與最新整合的【即時售罄告警 (instant_alert)】。
   - 【主力商品管理】：直覺的商品搜尋介面，可一鍵加入「主力列管商品」清單。支援含有雙引號規格（如 2"、3" 等英吋）的商品安全加入。

--------------------------------------------------
為了提供最佳的查價與盤點檢索體驗，本系統的搜尋引擎內建了以下進階最佳化技術：

1. 中文同音異體字/簡繁自動對應：
   系統在背景進行比對時，會自動將搜尋詞與商品資料庫中的常見異體字、簡體字進行雙向標準化映射。不論您或建檔人員寫的是哪一個，都能一次找出所有寫法相同的商品：

   - 接著採用「逆向分塊緩衝讀取」技術，在 1.3 GB 的 `ougo1.dbf`（銷貨明細）中由新到舊快速檢索。

3. 效能優化：
   系統僅讀取檔案末端最新資料，匹配滿 3 筆即自動終止，即使在 100Mbps 慢速網路共用磁碟下，查詢也能在 0.2 秒內完成，不影響 ERP 伺服器效能。

十、 庫存警戒水位與影子觀察模式（規劃落實中） 📊
--------------------------------------------------
為優化採購與庫存控制，系統規劃導入「動態安全庫存水位」影子觀察期（預計觀察 3 個月）：

1. ⚠️ **保障天數與回溯窗口分離**：
   - 系統預設提供 **10 個工作天** 的安全庫存銷量保障。
   - 計算日均銷量時，不採用 10 天短窗口（防劇烈跳動誤報），而是回溯過去 **90 天** 銷售紀錄以取得平滑、穩定的日均銷量。
2. 🔄 **交期自適應保護**：
   - 針對採購前置交期（Lead Time, LT）大於 10 天的品項，系統自動將水位天數調整為 `Max(10天, 交期 LT)`，防止交期過長造成斷貨真空期。
3. 💾 **資料庫零效能衝擊**：
   - 系統固定於 **凌晨 05:00** 離峰時間自動背景執行，拷貝銷貨資料完成計算後，寫入專用本機小檔 `shadow_level.json`。
