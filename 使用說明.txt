==================================================
         ERP 隨身助手 (Telegram Bot) 使用說明書
==================================================

本系統定位為針對傳統 VFP ERP 系統（以 DBF 為資料庫核心）的「零侵入式、智慧化行動外掛」。在完全不改動原廠 ERP 程式、不影響內勤日常作業的前提下，打通區網 ERP、雲端協作與業務行動端。

專案最高指導原則：
1. 核心 ERP 系統唯讀：本外掛讀取資料均採唯讀方式，嚴禁直接修改原始 ERP 核心資料表，確保資料庫安全與穩定。
2. 本機安全防護鎖：管理後台與敏感 API 僅限本機 IP (127.0.0.1 / localhost) 訪問，公網訪問會直接被阻斷。

==================================================
一、 系統檔案與 D 槽戰區架構
==================================================

系統已全面配置於 D 槽進行邏輯隔離：
- `D:\00_私人專區\01_外掛狀態小工具\`：存放小工具、備份指令檔與批次檔。
- `D:\01_唯讀備份\`：存放置頂的唯讀備份包，供白天查價對準此區，消除資料庫鎖定。
- `D:\03_AI_正式上線\`：正式生產線，Port 3000。

主要執行檔案清單 (D:\03_AI_正式上線\)：
1. vfp_sync_host.exe (Node.js 二進位背景服務)
   - 負責常駐背景執行查價機器人與設定網頁伺服器。
2. config.json (主設定檔)
   - 管理機器人 Token、核准功能與授權 Chat ID。
3. user_manual.md (手機查價使用說明書)
   - 提供給外勤業務的手機窄螢幕排版版使用手冊。

外掛狀態小工具清單 (D:\00_私人專區\01_外掛狀態小工具\)：
1. service_monitor_widget.ps1 (狀態小工具)
   - 提供高質感的桌面懸浮小視窗，實時監控 Bot 與 Web 服務狀態。
2. service_monitor_gui.ps1 (控制面板程式)
   - 提供完整的後台管理面板，點擊「重啟」或「結束」服務。
3. backup_daily.ps1 (手動與自動備份腳本)
   - 將 ERP 資料庫壓縮打包並存至 `D:\01_唯讀備份\`，保留最近 7 天備份，完成後彈出系統提示框。
4. run_manual_backup.bat (手動一鍵備份)
   - 雙擊即可直接以相對路徑安全呼叫備份腳本。

二、 外掛狀態小工具與 ERP 備份使用指引
==================================================

1. 開啟監控小工具：
   - 雙擊桌面上的「外掛狀態小工具」捷記，即可啟動懸浮在桌面右上角的小視窗。
   - 懸浮小工具會即時以綠色/紅色燈號標示 Bot (28256) 與 Web (3000) 的運作狀態。
2. 重啟與結束背景服務：
   - 在小工具上點選「🔄 重啟」按鈕，會非同步重啟正式區的所有服務進程。
   - 點選「🛑 結束」按鈕，可一鍵關閉並清理所有背景進程。
3. 立即執行 ERP 備份：
   - 在小工具或控制台點選「💾 備份」按鈕，按鈕會顯示「⏳ 備份中...」，此為非同步執行，完成後會彈出視窗提示「ERP 資料庫備份完成！備份檔已存放在 D:\01_唯讀備份\」。
   - 同時亦可雙擊 `D:\00_私人專區\01_外掛狀態小工具\run_manual_backup.bat` 一鍵執行備份。
   - 備份日誌會妥善儲存於 `D:\00_私人專區\01_外掛狀態小工具\logs\backup_daily.log`。

三、 查價與客戶歷史售價查詢 (Telegram Bot)
==================================================

1. 商品一般查價與庫存查詢：
   - 格式：在 Telegram 對話框直接輸入商品關鍵字，免空格，多關鍵字可用空格隔開。
   - 範例：`3m1吋`、`3m 膚色 1吋`、`酒精75`
   - 特色：
     * 支援同音異體字自動對應搜尋（如：溼↔濕、台 ↔ 臺、寸 ↔ 吋、尺 ↔ 呎）。
     * 商品排序依「有庫存 ➡️ 無庫存 ➡️ 已停用商品」三層分流置頂排序，優先呈現可售商品。
2. 客戶專屬歷史成交價查詢（防報錯價）：
   - 格式：同時輸入「客戶關鍵字」與「商品關鍵字」，用空格分開。
   - 範例：`大德藥局 3m`、`永和診所 酒精`
   - 回傳：系統自動使用記憶體客戶案號索引精準檢索 1.3GB 歷史銷貨明細檔，在 1-4 毫秒內極速回傳該客戶最近 3 次成交的日期、商品規格、購買數量與實際單價，不佔用 Z 槽 ERP I/O 且完全不鎖檔。
   - 即時回饋：查詢期間會自動觸發 Telegram「正在輸入 (typing...)」狀態；若點擊查詢結果的「歷史價」按鈕，畫面頂端會即時彈出「正在查詢歷史成交價...」提示氣泡，防止重複點擊。

四、 網頁設定後台說明 (Port 3000)
==================================================

1. 進入後台：
   - 在本機瀏覽器輸入：`http://lvh.me:3000/config` 即可進入後台。
   - 由於 Telegram 限制，使用 `lvh.me`（萬用本機解析網域）可在 Telegram 對話中以藍色超連結直接點選開啟。
2. 功能管理：
   - 可動態新增或刪除機器人插槽 (Slots)，並為每個機器人 Token 勾選啟用功能（查價 Query、歷史售價 History、盤點 Take、推播 Push）與綁定授權 ID。
   - 調整低庫存警戒值與每日定時推播時間。

五、 系統環境優化配置
==================================================

1. Chrome 遠端桌面防斷線設定：
   - 已關閉 Chrome 記憶體節省模式，並將 `remotedesktop.google.com` 納入白名單，開啟背景常駐執行，解決遠端桌面常態斷線痛點。
2. 區網點陣印表機 (EPSON LQ-690C) 設定：
   - 印表機 IP 為 `192.168.1.98` / `9100` 連接埠。
   - 驅動程式已升級為官方專屬的 `"EPSON LQ-690C ESC/P2"` 驅動程式，印單定位精確正常。
