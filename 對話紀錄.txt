二、 系統架構與檔案清單
--------------------------------------------------
所有檔案已完全整理至以下本機路徑。為維護安全，本系統不在 Z 槽 (NAS) 進行任何遠端程式備份。

主要本機執行路徑：C:\agy_Add_on\ERP_System\
1. telegram_bot.js (查價與歷史銷售及智慧推播機器人)
   負責 Telegram 查價與關鍵字客戶歷史成交價查詢與主動推播智慧警告與異常監控（背景執行）。
2. take_server.js (盤點與設定伺服器)。
   負責監聽本地 Port 3000 提供視覺化設定後台，並透過 Localtunnel 穿透至外網網址並主動推播異常與備份結果，帶有 API 金鑰保護機制。
3. config.json (統一設定檔)。
   支援多個機器人與智慧型警告、API 金鑰及資料庫設定。
4. stock_snapshot.json (庫存狀態快照檔)。
   智慧缺貨比對程式，用來記錄上次檢查之庫存數，防訊息重複發送。
5. append_take.ps1 (寫入腳本)。
   將盤點上傳之條碼、數量、主/次單位寫入，並非同步寫入 VFP 資料庫之 take.dbf 與 take1.dbf。
6. public/ (前端網頁資源資料夾)。
   提供條碼掃描行動端 UI，包含 index.html、index.css 與 index.js（含廠商模糊過濾選單）。
7. run_query.bat / run_query.vbs (查價背景啟動腳本)。
   用於手動/背景無字元視窗啟動 telegram_bot.js。
8. run_take.bat / run_take.vbs (盤點背景啟動腳本)。
   用於手動/背景無字元視窗啟動 take_server.js。
9. 自動開機啟動代理 (ERP_隨身助手.vbs)。
   置於啟動 Startup（開機自啟資料夾），開機自動執行並守護「查價」與「行動盤點網頁」服務。
10. run_bot.vbs (引導腳本)
    置於 C:\agy_Add_on\run_bot.vbs，防 Windows 警告彈出。

  - 主要商品低庫存警告：在 config.json 中新增 priorityProducts 欄位（主要商品編號陣列）。只要主要商品庫存低於預設門檻，便會寫入警報。

【第十二階段：API 金鑰安全鎖、盤點網站模糊查詢與設定後台功能擴充】

  - 外網 API 金鑰安全鎖：設定檔中隨機生成 22 位安全 Token。Telegram 發送的盤點網址中自動帶有該 Token，前端載入後存入 Session，隨後請求必須帶有此金鑰，否則拒絕連線。
  - 網頁端下拉選單模糊搜尋：前端加入過濾器，解決在手機端操作不便的問題。

  - Telegram 本機連線繞行：Telegram 官方不允許 localhost 超連結，故改為使用 lvh.me（萬用本機解析網域）繞過此防護。
