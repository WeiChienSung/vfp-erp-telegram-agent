一、 設計初衷與解決痛點 (Design Origin)
在傳統倉儲盤點作業中，盤點流程常面臨以下痛點：
1. 人工紙筆記錄：盤點人員拿紙張記錄條碼、手寫數量，回到電腦前再手動逐筆打入 ERP，耗費雙倍工時且打字極易出錯。
2. 條碼槍受限：傳統 USB 條碼槍必須連接在桌上型電腦或筆電旁，行動力低；無線條碼槍價格昂貴且無螢幕顯示當前商品名稱，無法即時核對。
3. 網路訊號中斷：倉庫深處常有 Wi-Fi 或 4G 斷訊問題，造成線上網頁版盤點系統經常斷線、資料遺失。
4. 併發鎖定衝突：寫入盤點資料至 VFP 資料庫時，若正好與 ERP 系統其他開單動作衝突，容易引發檔案鎖定失敗甚至造成資料表毀損。

本系統發想了「用手機當作條碼機，離線緩衝寫入，安全性預檢鎖」的輕量化 Web 方案。

二、 核心技術發想與優化細節 (Technical Innovations)
1. 📱 手機瀏覽器相機掃碼免安裝 (HTML5 Barcode Scan)
   前端網頁利用 HTML5 Camera API 配合開源解碼庫，直接調用手機主相機進行實時條碼掃描，免去下載安裝任何 APP 的繁瑣流程。
   
2. 💾 離線沙盒快取防斷網保護 (Local Session Cache)
   為防範倉庫訊號死角，盤點網頁內建離線沙盒暫存區。人員掃描並輸入數量後，資料會優先儲存於瀏覽器的 LocalStorage 中。在斷網狀態下仍可流暢盤點，待回到訊號良好區域後，一鍵上傳至伺服器。
   
3. 🛡️ 物理級本機設定防護與 API 安全鎖
   - 本地設定鎖：設定後台與 `/api/config` 僅能由本機主機 (127.0.0.1) 存取，防堵外網探針。
   - API 安全憑證鎖：外網盤點上傳 API 採用密碼學級安全 Token 驗證，並防範暴力破解。

4. 🔒 預檢鎖機制 (Pre-lock Check) 寫入保護
   為了解決 VFP 排他鎖定衝突，在盤點資料正式寫入暫存檔前，背景 PowerShell 腳本 (`append_take.ps1`) 會先嘗試同時鎖定並打開 `take.dbf`（單頭）與 `take1.dbf`（單身）。只有在確認雙檔案皆未被 ERP 進程獨佔鎖定時，才會開始執行寫入。如果遇到獨佔衝突，會自動返回錯誤並在背景重試，保障寫入的原子性，徹底杜絕 VFP 資料庫因中途鎖定衝突而發生「有單頭、無明細」的毀損狀態。

三、 暫存檔安全隔離原則
盤點上傳結果僅能寫入特定的盤點暫存資料表（`take.dbf` 及 `take1.dbf`），絕對不直接修改或覆蓋 ERP 的常規主資料表，物理上隔離了對運行中舊系統的影響，確保核心 ERP 穩定運作。
